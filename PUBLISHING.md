# How this repository was organized — and how to publish it

This document explains exactly what was done to turn the original project
folder into an open-source-ready repository, and gives step-by-step
instructions for putting it on GitHub (or any git host).

## 1. What was reorganized, and why

The original folder was a working Keil project plus an SDCC port and loose
build artifacts. It was restructured into a conventional firmware repo:

| Original | Now | Why |
|----------|-----|-----|
| `source/*.c, *.h` (mixed GBK/UTF-8, Chinese comments) | `firmware/keil/source/` — UTF-8, English comments | Open-source convention; the GBK-encoded `config.c` rendered as mojibake everywhere outside a Chinese-locale Windows machine. |
| `project/*.uvproj` + `STARTUP.A51` | `firmware/keil/project/` | Kept side-by-side with `source/` so the project's relative `..\source\` paths keep working. Renamed to `WS2812_STC8G1K08A.uvproj` (no spaces). |
| `project/*.uvopt`, `*.uvgui.*` | **dropped** (gitignored) | Per-user IDE state, regenerated by µVision. |
| `sdcc_build/*.c, *.h, build.sh` | `firmware/sdcc/` | The SDCC port, minus its build artifacts (`.rel`, `.lst`, `.asm`, …), which are gitignored and regenerated by `build.sh`. |
| `sdcc_build/*.hex` | `prebuilt/ws2812_stc8g1k08a_35mhz_brightness-rainbow.hex` | The one ready-to-flash binary, built from the current sources (brightness, rainbow mode + EEPROM memory), byte-for-byte reproducible. |
| `output/WS2812 STC8G1K08A.hex` | **dropped** | Stale: its build log (2025-03-04) shows it was compiled from an older source revision — `eeprom.c` was not even part of that build, so it lacks the settings-memory feature. Shipping it would hand users the wrong firmware. |
| `下载说明/下载选项.png` | `docs/images/stc-isp-settings.png` | The flashing-settings screenshot is documentation. |
| `下载说明/新建文本文档.txt` ("灯IO口接3.3 内部晶振35M") | folded into both READMEs | The two critical facts it contained (data pin = P3.3, internal oscillator = 35 MHz) are now documented properly. |
| `Clean.bat` | **dropped** | Superseded by `.gitignore`. |
| — | `LICENSE` (MIT), `README.md`, `README.zh-CN.md`, `.gitignore` | The open-source packaging itself. |

**Code was not changed.** Only comments were translated and encodings
normalized. Two verifications back this up:

1. Rebuilding `firmware/sdcc/` with SDCC 4.6.0 produces a hex **byte-identical**
   to the original `sdcc_build` hex.
2. Every Keil-tree file, stripped of comments and whitespace, is **identical**
   to its original — with one deliberate exception: `config.h` now includes
   `"STC8G.H"` with the correct upper-case filename (the original `"STC8G.h"`
   only worked on case-insensitive filesystems).

## 2. Before you publish — checklist

- [x] Copyright line in `LICENSE` is attributed to **LongXL**.
- [ ] Confirm you have the right to open-source all of it. The two bundled
      vendor files (`STC8G.H`, `STARTUP.A51`) are routinely redistributed in
      open-source 8051 projects and are flagged as third-party in `LICENSE`.
- [ ] Optional: put a product link (where to buy your finished board) into the
      "Where to buy" section of both READMEs.

## 3. Publish to GitHub — step by step

> **Status:** already done — the repo is live at
> <https://github.com/LongXL6/LongXL-LED-Open-Source-Version>
> (initially published as `ws2812-stc8g1k08a`, then renamed; GitHub redirects
> the old URL automatically). The steps below are kept for reference, e.g. if
> you ever mirror the repo elsewhere.

The local git repository is already initialized with an initial commit, so
publishing is just creating the remote and pushing.

### Option A — with the GitHub CLI (`gh`)

```sh
cd LongXL-LED-Open-Source-Version
gh auth login                        # once, if not logged in
gh repo create LongXL-LED-Open-Source-Version \
    --public \
    --source=. \
    --description "One-button WS2812/NeoPixel controller on a $0.27 STC8G1K08A (8-pin 8051): 7 colors, brightness ramp, EEPROM memory. Keil + SDCC." \
    --push
```

### Option B — with the GitHub web UI

1. Go to <https://github.com/new>.
2. Repository name: `LongXL-LED-Open-Source-Version`. Visibility: **Public**.
   Do **not** initialize with a README/.gitignore/license (this repo already
   has them).
3. Create, then push the existing repo:

```sh
cd LongXL-LED-Open-Source-Version
git remote add origin https://github.com/<YOUR-USER>/LongXL-LED-Open-Source-Version.git
git push -u origin main
```

### After the first push

```sh
# Tag the current state and create a release carrying the flashable binaries
git tag -a v1.0.0 -m "First open-source release"
git push origin v1.0.0
gh release create v1.0.0 prebuilt/*.hex \
    --title "v1.0.0" \
    --notes "First open-source release. Flash with STC-ISP at 35 MHz IRC (see README)."
```

On the repo page, add topics so people can find it:
`ws2812` `neopixel` `stc8g` `stc8` `8051` `mcs-51` `sdcc` `keil-c51`
`led-controller` `addressable-led`.

## 4. Suggested repo description (copy-paste)

> One-button WS2812/NeoPixel LED strip controller firmware for the 8-pin
> STC8G1K08A (1T 8051). Short press cycles 7 colors, long press ramps
> brightness, settings persist in EEPROM. Bit-banged 800 kHz protocol on the
> 35 MHz internal RC — no crystal, no timers. Builds with Keil C51 and SDCC.

## 5. Housekeeping conventions going forward

- Keep code comments and commit messages in English; docs can be bilingual
  (`README.md` + `README.zh-CN.md`).
- Never commit build artifacts; `.gitignore` already covers Keil and SDCC.
- If you change LED count, pins, or timing, update the Configuration table in
  **both** READMEs.
- New prebuilt binaries go into GitHub Releases (keep `prebuilt/` for just the
  single latest known-good build, and only replace it with a hex that was
  rebuilt from the committed sources).
